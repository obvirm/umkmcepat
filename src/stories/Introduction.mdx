UMKM Cepat uses Storybook as the live design-system catalog. Use this before changing UI so new work follows the same typography, spacing, color, radius, and interaction patterns.

## Agent workflow

1. Read `DESIGN.md`.
2. Open the closest Storybook story.
3. Reuse existing primitives and product patterns.
4. Add or update a story when adding a reusable component or visual state.
5. Run `bun run check`, `bun run storybook:build`, and `bun run test:storybook`.

## What belongs here

- Reusable primitives: buttons, dialogs, form controls.
- Product UI states: login, profile, project list, workspace cards, generated preview.
- Important responsive variants.
- Loading, disabled, empty, destructive, and error states.

## What does not belong here

- One-off decoration.
- Server-only logic.
- Fake dashboard art that does not exist in the product.
- Components created only because “maybe later”.

## Visual rules

- Plus Jakarta Sans everywhere.
- Warm neutral base: `#eceae4`, `#fcfbf8`, `#f7f4ed`.
- Primary action: `#1c1c1c`.
- Product dark surfaces use `#151515` with warm-white text and low-opacity borders.
- Depth comes from borders and restrained surfaces, not heavy shadows.

## Adding a story

Create `*.stories.tsx` near the component or under `src/stories`.

```tsx
const meta = {
  component: Button,
  tags: ["autodocs"],
  title: "Atoms/Button",
} satisfies Meta<typeof Button>;
```

Use stories as examples and tests. If the component has behavior, add a `play` function.
